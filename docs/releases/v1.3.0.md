# Skill Base Template v1.3.0

This release mirrors the descendant repository's community and sponsorship setup on the template, and strengthens the bootstrap instructions for transmitting goals and values and for confirming alignment with the user before a skill is built.

## Highlights

- Enrich `.github/FUNDING.yml` with documented sponsorship platform examples.
- Mirror the descendant's community setup: Discussions enabled, Projects and Wiki disabled.
- Require a core-principles-and-decision-stance section in the generated `AGENTS.md`.
- Add an alignment-confirmation step to the exploratory phase, before skill construction.
- Gate construction on the confirmed design synthesis, with a matching must-follow rule in `AGENTS.md`.

## Why It Matters

The template now demonstrates the same community and sponsorship configuration it produces, so a generated repository inherits a setup that has been seen working upstream.

The bootstrap improvements come from reviewing the descendant's generated `AGENTS.md`. Its core-principles section and its decision stance, surface findings as obstacles to the goal, weigh local context, and bring decisions to the user, are how goals and values reach future agents that must make local decisions. The bootstrap guidance now requires this in every generated `AGENTS.md` rather than leaving it to chance.

The exploratory phase previously moved from intake adequacy straight into construction. When the readiness decision rested on the agent's own inferences, the skill could start misaligned with the user's intent. The new alignment-confirmation step has the agent present its synthesized understanding for confirmation or correction before it builds, while staying clear of pushing design work back to the user.
